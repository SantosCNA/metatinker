# 信息架构（IA）基线

> 版本：v0.1（2025-11-13）  
> 维护人：元像AI Lab（MetaTinker）

## 站点结构

```
/
├── #hero               # 着陆英雄区
├── #products           # 作品矩阵（6 个项目）
├── #services           # 专业服务
├── #workflow           # 合作流程
├── #insights           # 洞察与研究
├── #contact            # 联系与预约
└── /blog               # 技术研究（计划中）
```

### 页面分层

- **首页（/）**  
  强化品牌形象与快速价值传达，包含 Hero、价值主张、作品矩阵、服务速览、洞察卡片、联系 CTA。
- **服务页面（/services）**  
  展示两大核心服务「AI 系统架构咨询」「AI 解决方案设计」，附流程、交付物、责任边界。
- **案例页面（/cases）**  
  深度拆解代表项目：背景 / 方案 / 技术贡献 / 结果 / 技术栈。
- **关于页面（/about）**  
  展现工作室定位、方法论、质量承诺与合规说明。
- **博客页面（/blog）**  
  技术洞察、架构实践、解决方案专题、技术难点突破（MDX 方案，二期上线）。

## 内容模块映射

| 模块              | 数据来源 / 维护方式              | 目标用户                 | 说明 |
| ----------------- | --------------------------------- | ------------------------ | ---- |
| Works Grid        | `src/data/products.ts`            | 投资人 / 潜在客户        | 使用状态、CTA、技术标签 |
| Services Listing  | `src/data/services.ts`            | 企业决策者               | 包含流程、交付成果 |
| Case Studies      | `src/data/cases.ts`               | 技术评审 / 决策层        | 支持标签过滤 |
| Insights Preview  | 手动选摘 + AI Insight Assistant   | 行业关注者 / 订阅用户    | 每月更新 3-4 条 |
| Blog Highlights   | `/blog/*.mdx`                     | 技术社区 / 猎头          | 自动抓取最新文章标题 |
| Contact CTA       | Formspree / Netlify Forms         | 商务 & 合作意向          | 包含责任边界提示 |

## 内容语调指引

- 采用「个人工作室」视角，避免夸大团队规模。  
- 强调「研究 + 方案设计 + 咨询指导」，明确非外包开发。  
- 避免敏感行业承诺，聚焦技术深度与方法论。  
- 中英文术语并列（如有），保持国际化语境。
