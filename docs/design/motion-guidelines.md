# 动效与交互规范

> 版本：v0.1（2025-11-13）  
> 维护人：元像AI Lab（MetaTinker）  
> 说明：动效以 Framer Motion 实现为参考，Figma 原型中提供了关键帧示例。

## 1. 全局动效设定

- **Ease 函数**：`cubic-bezier(0.4, 0, 0.2, 1)`（默认），`ease-out` 用于元素进入动画  
- **时长约定**：
  - 微交互：`200ms`（按钮、标签、表单元素）  
  - 列表/卡片：`240ms`（含延迟）  
  - 页面过渡：`300-400ms`  
  - 背景粒子动画：`8-12s` 循环缓动
- **延迟策略**：同一组元素最多错峰 `120ms`，保证节奏层次。

## 2. 关键场景动效

### Hero 区域

- **标题进入**：自下而上 (`translateY: 24px → 0`)，透明度 `0 → 1`，时长 `360ms`  
- **标签列表**：每个标签延迟 `40ms`，缩放 `0.9 → 1`，透明度 `0 → 1`  
- **CTA 按钮**：初始淡入 + 渐变光晕扩散（时长 360ms，延迟 160ms）

### 价值主张卡片

- 初始：缩放 `0.94 → 1`，透明度 `0 → 1`，时长 `320ms`  
- Hover：Y 轴上移 `4px`，阴影增强，描边颜色过渡 `200ms`

### 轮播/案例展示

- 列表滚动：采用 `framer-motion` 的 `viewport` 滚动检测，出现时淡入  
- 卡片 hover：添加微小视差（背景渐移），阴影 `shadow-glow`，时长 `220ms`  
- “查看全部”链接：下划线从左向右展开（`scaleX: 0 → 1`，`180ms`）

### 微信/联系模块

- 背景光效呼吸：透明度 `0.3 ↔ 0.45`（8s 循环），可使用 CSS keyframes  
- 二维码容器：悬停时发光 `shadow-glow` + 轻微缩放 `1.0 → 1.03`

## 3. 状态反馈

- 表单输入：聚焦时描边颜色转为 `brand.primary`，阴影 `0 0 0 2px rgba(99, 102, 241, 0.25)`  
- 校验错误：边框切换为 `accent-amber`，文字提示淡入，时长 `180ms`  
- 提交成功：按钮进入加载态（旋转图标 600ms），完成后切换为成功提示。

## 4. 质感增强

- **玻璃拟态层**：使用 `backdrop-blur-xl`，透明度 `0.6`，hover 时加深背景色 `+5%`  
- **霓虹光效**：聚焦区域（按钮、标题）添加 `drop-shadow`，控制不超过 `12px` 光晕  
- **粒子动画**：SVG 或 Canvas 粒子缓慢漂浮，随机路径振幅 `12-20px`

## 5. 性能策略

- 优先使用 CSS 动画和 transform，避免频繁触发布局重排  
- 复杂动效（粒子、背景）在移动端降级：减少粒子数量、关闭大面积模糊  
- Ensure 动画可被用户设置 `prefers-reduced-motion` 时禁用。

## 6. 实现参考

- **Framer Motion 模板**：

```tsx
const fadeUp = {
  hidden: { opacity: 0, y: 24 },
  visible: { opacity: 1, y: 0, transition: { duration: 0.36, ease: [0.4, 0, 0.2, 1] } },
}
```

- **CSS 呼吸动画**：

```css
@keyframes glowPulse {
  0% { opacity: 0.3; }
  50% { opacity: 0.45; }
  100% { opacity: 0.3; }
}
```
